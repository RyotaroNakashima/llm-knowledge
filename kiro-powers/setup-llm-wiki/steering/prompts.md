# LLM Wiki セットアッププロンプト

## フルセットアッププロンプト

```text
このプロジェクトに LLM Wiki をセットアップしてください。

目的:
- Kiro / Codex / Claude Code などの LLM エージェントが、プロジェクトの長期記憶を読める・更新できるようにする。
- 人間も Markdown と必要なら Obsidian で読めるようにする。
- Obsidian は optional とし、入れられない環境でも使える構成にする。

まず現在のフォルダ構成を確認してください。
そのうえで、必要最小限の Markdown ベースの Wiki 構造を作ってください。

推奨構成:
- AGENTS.md
- index.md
- 00_受信箱/
- 10_情報源/
- 20_ノート/
- 30_意思決定/
- 40_プロジェクト/
- 80_テンプレート/
- 90_運用/

作成してほしいもの:
- AGENTS.md: LLM エージェントが作業前後に従う入口ルール。作業前に読むもの、作業後の更新判断、更新先、作業ログを全文保存しない方針、トークン節約方針を含める
- index.md: 入口、現在のテーマ、主要ノート、未解決事項
- 90_運用/運用マニュアル.md: フォルダの役割、日常ワークフロー、ノート基準、保守方法
- 90_運用/LLM記憶プロトコル.md: LLM が最初に読むもの、更新ルール、残す情報、残さない情報、作業後プロンプト
- 40_プロジェクト/<プロジェクト名>.md: 現在の状態、重要な文脈、次の行動
- 30_意思決定/YYYY-MM-DD-LLM記憶にMarkdownを使う.md: Markdown を使う決定、理由、却下案、結果
- 80_テンプレート/概念ノート.md
- 80_テンプレート/意思決定ノート.md
- 80_テンプレート/プロジェクトノート.md

重要:
- チャット全文やログ置き場にしないでください。
- 将来の判断や作業を変える情報だけを長期記憶として圧縮してください。
- すべての作業を自動記録する運用にはしないでください。意味のある変更があった場合だけ関連ノートへ統合してください。
- 既存のコードや文書を壊さないでください。
- 作成後にファイル一覧、リンク、JSON があれば JSON 構文を検証してください。
```

## Markdown-only 会社環境向けプロンプト

```text
会社環境で Obsidian が使えない前提で、Markdown-only の LLM Wiki をセットアップしてください。

Obsidian 固有機能や .obsidian 設定は作らないでください。
ただし、将来 Obsidian や別エディタで開いても困らないように、標準 Markdown と相対リンクで構成してください。

LLM エージェントが使いやすいように、AGENTS.md、index.md、90_運用/LLM記憶プロトコル.md を必ず作ってください。
検索は rg、編集は通常のファイル編集、履歴管理は Git を前提にしてください。
```

## 作業後の長期記憶抽出プロンプト

```text
この作業から、将来の LLM エージェントや自分が再利用すべき長期記憶を抽出してください。

既存の Wiki を読み、必要なら既存ページを更新し、新規ページは最小限にしてください。
事実、判断、根拠、却下した案、未解決事項、次回の注意点を分けて書いてください。
古くなりそうな情報には日付と信頼度を付けてください。
チャット全文や一時的な作業ログは保存しないでください。
```
